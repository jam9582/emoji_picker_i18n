## 2026-07-09 ② 데이터 파이프라인 (`tool/generate_data.dart`)

- **소스**: emojibase-data 16.0.3 compact.json (jsDelivr CDN에서 다운로드)
- **필터**: group이 없거나(국기 문자 조각 등 26개) group 2(피부색 부품 9개)인 항목 제외 → **피커 대상 1,906개**. 전부 order·tags 보유 확인
- **출력 형식** ("공통 1벌 + 언어별 키워드" 설계 구현):
  - `lib/src/data/emoji_common.dart`: `'unicode|group'` 또는 `'unicode|group|skin1,skin2,...'` — 표시 순서(order)로 정렬, 59KB
  - `lib/src/data/emoji_locale_<xx>.dart`: `'label|tag1|tag2|...'` — 공통과 같은 인덱스, ko 127KB / en 103KB
  - `lib/locales/<xx>.dart`: 사용자용 공개 import 경로 (내부 파일 재export)
- **인덱스 정합성**: 공통은 en 기준으로 만들고 각 언어는 hexcode로 매칭 (언어 간 순서 차이 방어)
- **파이프라인 안에서 배운 것**: 구분자 `|`가 데이터에 있으면 즉시 예외 — 침묵 손상 방지
- 다운로드 캐시(`tool/cache/`)는 재생성 가능하므로 gitignore

### 여기서 발견한 함정: 이모지 표시 지정자(U+FE0F)

- 테스트에서 `contains('🐈')`가 실패 → 데이터의 🐈는 투명한 "컬러로 그려라" 지시문자(U+FE0F)가 붙은 2글자였음
- 원인·해결법 후보·적용 범위는 내부 기획 문서에 상세 기록. 결론: 데이터는 원본 유지, 출처가 다른 문자열이 만나는 지점(선택값 강조·역조회·최근사용 중복 제거)에서만 정규화

## 2026-07-09 ③ 한글 초성·자모 유틸 (`lib/src/search/hangul.dart`)

- 한글 음절은 `0xAC00 + 초성×588 + 중성×28 + 종성` 공식으로 분해 → **사전 데이터 0바이트**
- 두 가지 표현을 생성:
  - 초성 문자열: '고양이' → 'ㄱㅇㅇ' (초성만 친 검색어용)
  - **자모 스트림**: '고양' → 'ㄱㅗㅇㅑㅇ' (그 외 모든 한글 검색어용)
- 매칭 규칙: 검색어가 초성으로만 구성 → 초성 문자열 포함 검사 / 그 외 → 양쪽을 자모 스트림으로 펴서 부분 문자열 검사
- **자모 스트림 방식의 보너스**: '고ㅇ'(ㄱㅗㅇ)과 타이핑 중간 상태 '공'(ㄱㅗㅇ)이 같은 스트림이 되어, 한글 입력기의 중간 조합 상태에서도 자연스럽게 검색됨. 별도 처리 없이 공짜로 얻은 기능
- 겹받침(값→ㄱㅏㅂㅅ)·겹모음(화→ㅎㅗㅏ)은 키보드 타이핑 순서로 분해 — 양쪽을 같은 규칙으로 펴기 때문에 정합성 유지

## 2026-07-09 ④ 검색 엔진 (`lib/src/search/emoji_search.dart`)

- 구조: 생성 시 색인 1회 구축(키워드 소문자화 + 한글이면 초성·자모 스트림 캐시) → 검색은 선형 스캔이지만 1,906×키워드 수 규모라 1ms 미만
- 여러 언어를 넘기면 **모든 언어 동시 검색** (kEmojiLocaleKo + kEmojiLocaleEn → '고양이'도 'cat'도 매칭)
- `findByChar()`: 정규화된 문자열 → Emoji 역조회 맵. 외부 저장값 강조 표시와 피부색 역조회 담당
- `normalizeEmoji()` / `sameEmoji()`: FE0F/FE0E 제거 비교 유틸을 공개 API로 노출

### 순위 체계 결정

- 첫 구현에서 '고양이' 검색의 1등이 🐈(이름=고양이)가 아니라 😺(태그에 고양이)로 나오는 문제 발견
  - 원인: 이름 정확 일치와 태그 정확 일치가 동점 → 표준 순서상 앞인 😺가 승리
- 결정한 순위: **정확 > 접두 > 포함의 3단계 × 각 단계에서 이름 > 태그** = 6단계
  (짝수=이름, 홀수=태그 점수로 구현. 동점이면 유니코드 표준 순서)
- 결과: '고양이' → 🐈(정확·이름) → 😺😸...(정확·태그) → '고양이 얼굴'(접두·이름) → '검은 고양이'(포함·이름) 순

### 테스트 현황

- 28개 통과: 데이터 정합성 5 + 한글 유틸 8 + 엔진 15 (포함·초성·혼합·치다만·순위·정규화·역조회)

## 2026-07-09 ⑤ 피커 UI 방향 결정 (기존 생태계 관행 조사 후)

기존 Flutter 이모지 피커들의 UI 관행을 조사하고 우리 방향을 확정했다.

**채택 (생태계의 검증된 관행):**
- 부위별 Config 패턴 — 설정을 search/category/grid/skinTone/recents 서브 컨피그로 분할. 기존 피커 사용자에게 익숙한 모양이라 이주 비용이 낮다
- 기본 뷰 + 빌더로 통째 교체 패턴 — 완전 커스텀 UI를 원하는 사용자용 탈출구
- 피부색 선택은 셀 롱프레스 → 셀 위 오버레이 팝업 (모바일 키보드들과 동일한 UX)
- 최근 사용에는 피부색을 벗긴 기본형으로 저장

**다르게 (우리 차별점):**
- 검색을 별도 화면 전환이 아니라 **상시 노출 검색창 + 그리드 실시간 필터**로 — 검색이 이 라이브러리의 간판이므로 치는 즉시 좁혀지는 경험이 우선
- 최근 사용 저장소를 인터페이스로 주입 (기본 구현만 shared_preferences) — 특정 저장소 의존 강제 회피, 테스트 용이
- 최근 사용 중복 판정은 `sameEmoji()` 정규화 비교 — 원시 문자열 == 비교는 표기 변형(FE0F)에 취약
- 카테고리를 enum 하드코딩 대신 데이터(group 번호) 주도로

**자체 네이티브 코드 없이 가기로 결정:**
- 구형 기기의 미지원 이모지(□) 필터를 네이티브 코드 질의로 하면 패키지가 플러그인이 되어 6개 플랫폼 빌드 복잡도가 올라감
- 대신 이모지별 유니코드 버전 메타데이터로 필터 (파이프라인에 version 필드 추가 예정 — emojibase 전체판 data.json에만 있어 소스 병합 필요)
- 제조사별 폰트 편차까지는 못 잡는 한계 인정. 필요해지면 네이티브 글리프 체크를 선택 설치형 부가 패키지로 제공
- 채팅 입력창 연동(TextEditingController)은 v1.0 범위에서 제외 — 1차 시장은 "이모지 하나 고르기"

## 2026-07-09 ⑥ 최근 사용 저장소: 생태계 관행 조사 후 결정

- 최근 사용 이모지를 영구 저장하려면 저장 수단이 필요한데, 처음엔 "플러그인 의존 없이 가볍게"를
  이유로 인터페이스만 제공하고 기본은 메모리 저장(앱 끄면 소멸)으로 구현했다
- **생태계 관행 조사** (pub.dev API로 의존성 실측): 주요 이모지 피커 패키지들은 전부
  shared_preferences 를 직접 의존하고, 유명 패키지들도 플러그인 의존(url_launcher 등)을
  일반적으로 한다. shared_preferences 는 Flutter 공식 팀 관리(flutter/packages), 6개 플랫폼
  전부 지원이라 빌드 리스크가 사실상 없음
- **결정**: 인터페이스(`RecentEmojiStorage`) 주입 구조는 유지하되, shared_preferences 기반
  `PrefsRecentEmojiStorage` 를 내장해 기본값으로 사용. 개발자는 0줄로 영구 저장을 얻고,
  자기 DB에 넣고 싶으면 인터페이스 구현을 주입
- 이 과정에서 초기의 "순수 Dart 유지" 원칙은 실익이 불분명해 폐기. 실질 기준으로 재정의:
  **자체 네이티브 코드를 새로 만들어 유지보수 부담(OS 업데이트 대응·플랫폼별 구현·네이티브
  디버깅)을 지지 않는다.** 공식 관리되는 표준 플러그인 의존은 그 부담이 남의 몫이므로 자유

### 최근 사용 동작 규칙

- 피부색 변형(👋🏽) 선택 시 기본형(👋)으로 저장 — `findByChar()`가 피부색 문자열도
  기본형으로 역조회하도록 색인 확장
- 중복 판정은 `sameEmoji()` 정규화 비교, 최신이 맨 앞, recentsLimit(기본 28) 초과 시 밀어냄
- 최근 탭을 보는 중에는 순서를 즉시 바꾸지 않음 (눈앞에서 이모지가 자리를 옮기는 혼란 방지)
- 저장 형식은 이모지 문자열 목록만 — 이름·피부색 메타데이터는 데이터에서 재조회 (데이터
  버전이 갱신돼도 저장값이 낡지 않음)
